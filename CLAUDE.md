## Project Overview

A fullstack Weather API wrapper service built with FastAPI (Python) and React + TypeScript (Vite). The backend implements Hexagonal Architecture (Ports & Adapters) with Clean Code principles. Frontend uses Glassmorphism design with animated SVG weather icons.

## Architecture

### Backend (FastAPI + Python)

The backend follows Hexagonal Architecture with clear separation of concerns:

```
backend/src/
├── domain/          # Core business logic (entities, ports)
│   ├── entities.py  # WeatherData dataclass
│   └── ports.py     # WeatherRepository abstract interface
├── application/     # Use cases (orchestration layer)
│   └── use_cases.py # GetWeatherUseCase with dependency injection
├── infrastructure/  # External adapters (API clients)
│   └── weather_api_adapter.py  # VisualCrossingAdapter
└── interface/       # HTTP layer (FastAPI routers)
    └── weather_router.py
```

**Key Pattern:** The UseCase receives a `repository: WeatherRepository` via constructor injection, keeping business logic decoupled from external APIs.

### Frontend (React + TypeScript + Vite)

```
frontend/src/
├── App.tsx         # Main component with search and weather display
├── App.css         # Glassmorphism styles and animations
├── main.tsx        # Entry point
└── assets/         # Static assets
```

**Key Features:**
- Animated SVG weather icons (sun with rotating rays, rain animation)
- Portuguese translation mapping for weather conditions
- Hardcoded API URL: `http://127.0.0.1:8000/weather/${city}`

## Common Commands

### Backend

```bash
cd backend

# Setup (if not done)
python -m venv ../venv
source ../venv/bin/activate
pip install -r requirements.txt

# Run development server (from repo root or backend/)
uvicorn main:app --reload --host 127.0.0.1 --port 8000

# Run with venv activation
source venv/bin/activate && uvicorn main:app --reload
```

### Frontend

```bash
cd frontend

# Install dependencies (if not done)
npm install

# Run development server
npm run dev

# Build for production
npm run build

# Run ESLint
npm run lint

# Preview production build
npm run preview
```

### Full Development Workflow

```bash
# Terminal 1 - Backend
cd backend && source ../venv/bin/activate && uvicorn main:app --reload

# Terminal 2 - Frontend
cd frontend && npm run dev
```

## Environment Configuration

Create `backend/.env`:
```env
WEATHER_API_KEY=your_visual_crossing_api_key
WEATHER_API_BASE_URL=https://weather.visualcrossing.com/VisualCrossingWebServices/rest/services/timeline
```

**Note:** The repository currently has `.env` committed with a hardcoded API key. In production, this should be moved to `.env.example` and `.env` added to `.gitignore`.

## API Endpoints

- `GET /health` - Health check
- `GET /weather/{city}` - Get weather data for a city
- `GET /docs` - Swagger UI documentation (auto-generated by FastAPI)

## Adding New Features

### Backend: Adding a New Use Case
1. Define any new entities in `domain/entities.py`
2. Add port methods to `domain/ports.py` if needed
3. Implement in adapter: `infrastructure/weather_api_adapter.py`
4. Create use case in `application/use_cases.py` with dependency injection
5. Expose via router in `interface/weather_router.py`

### Frontend: Adding Weather Data Fields
1. Update `WeatherData` interface in `App.tsx`
2. Ensure backend response includes the field
3. Add display logic in the JSX
4. Add styles to `App.css` if needed

## External API

The application integrates with **Visual Crossing Weather API**:
- Timeline endpoint: `{base_url}/{city}`
- Returns current conditions and forecast data
- Free tier available at visualcrossing.com
